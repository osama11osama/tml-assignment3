## What the papers teach (applied to this assignment)

### 1. Goodfellow et al. — *Explaining and Harnessing Adversarial Examples* (FGSM)

- Adversarial examples arise from the **linearity** of high-dimensional models — small ℓ∞ perturbations can flip predictions.
- **FGSM** is a one-step attack: fast to generate, good starting point for training.
- **Lesson for us:** a single FGSM step inside the training loop (FGSM-AT) is a cheap first robustness boost, but weaker than multi-step PGD.

### 2. Madry et al. — *Towards Deep Learning Models Resistant to Adversarial Attacks* (PGD + adversarial training)

- **PGD** approximates the strongest adversary within an ℓ∞ ball (projected gradient steps + random start).
- **Robust optimization:** train on adversarial examples generated by **maximizing loss**, not on clean images only.
- **Lesson for us:** **PGD adversarial training (PGD-AT)** is the main method to improve leaderboard robust score. Use ε = 8/255 (assignment standard), multi-step PGD during training.

### 3. Athalye et al. — *Obfuscated Gradients Give a False Sense of Security*

- Defenses that **block or distort gradients** (input masking, randomization tricks without AT) look robust to weak attacks but fail under **adaptive** attacks.
- **Lesson for us:** do **not** rely on preprocessing tricks alone. Use **proper adversarial training**. Evaluate locally with **strong PGD** (many steps), not only FGSM.
- Tutorial 6: server attack parameters are **hidden** — optimize for **general** robustness, not overfitting to one local PGD config.

## What NOT to do (Obfuscated Gradients + course rules)

| Avoid | Why |
|-------|-----|
| Input denoising / JPEG / bit-depth reduction only | Gradient masking — fails adaptive attacks |
| Tune only to local PGD-5 | Server uses different attack — overfitting |
| Submit every experiment | 60 min cooldown; use local eval first |
| resnet50 without need | Slower; try resnet18 AT first, then resnet34 if underfitting |
